# 🌐 GetWeb.ai — AI-Powered Website Builder

A full-stack web application that lets users **describe a website in plain English** and instantly get a **production-ready, responsive, single-page website** generated by AI. Iterate on designs through a conversational chat interface, edit code live with Monaco Editor, and deploy with one click. Built with **React 19, Express 5, MongoDB, and DeepSeek AI** (via OpenRouter).

## Overview

GetWeb.ai is an end-to-end platform that turns natural language descriptions into fully functional websites. Users sign in with Google, describe what they want, and the AI (DeepSeek Chat via OpenRouter) generates a complete HTML/CSS/JS single-page application — mobile-first, responsive, with SPA navigation, animations, and real images from Unsplash. Users can then refine their website through a chat-based editor, directly edit the source code with Monaco Editor, preview changes in real-time, and deploy their site to a shareable public URL — all from a sleek, dark-themed interface.

## Features

### 🔐 Authentication

- **Google Sign-In** — One-click authentication via Firebase Google popup.
- **Secure Sessions** — JWT tokens stored as HTTP-only cookies with 7-day expiry.
- **Protected Routes** — Dashboard, Editor, and Generate pages require authentication.
- **Auto Session Restore** — On page reload, the app fetches the current user from the backend and restores Redux state.

### 📊 Dashboard

- **View All Websites** — A responsive grid of cards, each with a live iframe thumbnail preview, title, and last-updated timestamp.
- **Create New Website** — Click "New Website" to navigate to the AI generation page.
- **Open Website Editor** — Click any card to open the full editor with chat, preview, and code panels.
- **Deploy & Share** — One-click deployment generates a slug-based public URL. Share deployed sites with a single click (copies link to clipboard).

### ✨ AI Website Generation

- **Natural Language Input** — Describe your website in a textarea (e.g., "A portfolio site for a photographer with a dark theme and gallery grid").
- **Intelligent Prompt Engineering** — Your description is wrapped in a master prompt instructing the AI to act as a "Principal Frontend Architect" producing:
  - Single-file, self-contained HTML/CSS/JS
  - Mobile-first responsive design with media queries
  - SPA navigation (Home / About / Services / Contact)
  - Real images from Unsplash
  - System fonts only (no external dependencies)
  - Smooth animations and transitions
  - Form validation
  - iframe-compatible output
- **Progress Indicator** — A 9-phase animated progress bar shows status during generation (Analyzing → Designing → Writing HTML/CSS → Final checks).
- **Retry Logic** — Up to 2 automatic retries if the AI response fails to parse.
- **Auto-Redirect** — On success, automatically navigates to the Editor page.

### ✏️ Conversational Website Editor

- **Chat Interface** — A full chat panel with message bubbles showing the conversation between you and the AI.
- **Iterative Refinement** — Each message sends your change request along with the current website code to the AI, which returns an updated version.
- **Animated Thinking Indicator** — A 12-step animated "thinking" indicator cycles through steps like "Understanding your request", "Planning layout changes", "Applying animations", etc.
- **Live Preview** — An iframe on the right panel renders the website in real-time using blob URLs. Updates instantly when the AI returns new code.
- **Monaco Code Editor** — View and directly edit the raw HTML/CSS/JS source code with syntax highlighting (`vs-dark` theme). Changes propagate to the live preview.
- **Full-Screen Preview** — Expand the preview to full screen with a single click.
- **Mobile-Responsive** — On smaller screens, the chat slides up as a full-screen overlay panel.

### 🚀 Deployment & Sharing

- **One-Click Deploy** — Generates a unique slug from the website title and makes it publicly accessible.
- **Public Live Site** — Anyone with the link can view the deployed website at `/live-site/:slug` — no login required.
- **Share Button** — After deployment, copies the shareable URL to clipboard with a toast notification.

### 💳 Credits & Billing

- **Credit System** — Every user starts with **100 free credits**.
  - Generating a new website costs **50 credits**.
  - Each AI update/refinement costs **25 credits**.
- **Stripe Checkout** — Upgrade to paid plans via Stripe payment (INR currency).
- **Webhook Integration** — Stripe webhooks automatically update user credits and plan upon successful payment.
- **3 Plans Available:**
  - **Free** — ₹0, 100 credits
  - **Pro** — ₹499, 100 additional credits (Most Popular)
  - **Enterprise** — ₹1,499, 2,000 credits

### 🎨 UI & Design

- **Dark Theme** — Sleek dark gradient background throughout the application.
- **Framer Motion Animations** — Smooth entrance animations, hover effects, and page transitions.
- **Lucide React Icons** — Consistent, clean iconography across the entire UI.
- **Toast Notifications** — Non-intrusive dark-themed toast messages for success/error feedback.
- **Responsive Design** — Fully responsive with Tailwind CSS v4, optimized for desktop and mobile.

## Getting Started

### Prerequisites

- **Node.js** v18+ and **npm**
- **MongoDB** (local instance or [MongoDB Atlas](https://www.mongodb.com/atlas))
- **OpenRouter API Key** — [Get one here](https://openrouter.ai/keys) (for DeepSeek AI access)
- **Firebase Project** — [Create one here](https://console.firebase.google.com/) (for Google Authentication)
- **Stripe Account** — [Sign up here](https://dashboard.stripe.com/register) (for payment processing)

### Backend Setup

```bash
# 1. Navigate to the backend directory
cd backend

# 2. Install dependencies
npm install

# 3. Create your environment file
cp .env.example .env

# 4. Fill in your actual values in .env (see Environment Variables section below)

# 5. Start the development server
npm run dev
```

The backend will run on **http://localhost:3000** by default.

### Frontend Setup

```bash
# 1. Navigate to the frontend directory
cd frontend

# 2. Install dependencies
npm install

# 3. Create your environment file
cp .env.example .env

# 4. Update VITE_BACKEND_URL if your backend runs on a different port

# 5. Start the development server
npm run dev
```

The frontend will run on **http://localhost:5173** by default.

## AI Website Generation

GetWeb.ai uses the **DeepSeek Chat** model via the **OpenRouter API** to generate websites. Here's how the pipeline works:

### Initial Generation (50 credits)

1. **User Input** — User describes their website in plain English.
2. **Prompt Engineering** — The description is embedded in a comprehensive master prompt that instructs the AI to:
   - Act as a "Principal Frontend Architect"
   - Generate a single self-contained HTML file with embedded CSS and JavaScript
   - Use mobile-first responsive design with breakpoints at 480px, 768px, 1024px, and 1280px
   - Include SPA-style navigation with 4 sections (Home, About, Services, Contact)
   - Use real images from Unsplash (via `source.unsplash.com`)
   - Add smooth CSS transitions and scroll animations
   - Include form validation
   - Use system fonts only (no external CDN dependencies)
3. **AI Response** — The model returns raw JSON: `{ "message": "...", "code": "<full HTML>" }`
4. **JSON Extraction** — The response is cleaned (strips markdown fences) and parsed.
5. **Database Storage** — A new `Website` document is created with the code, conversation history, and user reference.

### Iterative Updates (25 credits each)

1. **User sends a change request** via the chat input (e.g., "Make the header blue and add a testimonials section").
2. **Update Prompt** — The current website code + change request are combined into an update prompt.
3. **AI returns** the full updated codebase as JSON.
4. **Website is updated** — `latestCode` is replaced and the conversation is appended.

### Retry Logic

- If the AI response fails to parse as valid JSON, the system retries up to **2 additional times** before returning an error.

### Credit Costs

| Action | Cost |
|---|---|
| Generate a new website | 50 credits |
| AI-powered update/refinement | 25 credits |

### Payment Flow

1. User selects a plan on the Pricing page.
2. Backend creates a Stripe Checkout session with the plan details.
3. User completes payment on Stripe's hosted checkout page.
4. Stripe sends a `checkout.session.completed` webhook to the backend.
5. Webhook handler reads `userId`, `credits`, and `plan` from session metadata.
6. User's credits are incremented and plan is updated in the database.
